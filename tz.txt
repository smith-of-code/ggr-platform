ТЕХНИЧЕСКОЕ ЗАДАНИЕ 

РАЗДЕЛ 1. НАИМЕНОВАНИЕ ЛИЦЕНЗИИ

Предоставление неисключительной лицензии на использование программы для ЭВМ, для мероприятия «Туристическая онлайн-академия Росатома» в срок с 12.05.2025 г. по 20.12.2025 г.

Подраздел 2.1 Состав (перечень)  лицензии Общие требования 
Предоставление неисключительной лицензии на использование программы для ЭВМ, для мероприятия «Туристическая онлайн-академия Росатома» (далее – Система). 
Конфигурирование программы для ЭВМ, для АНО «Энергия развития» (далее – Заказчик) в соответствии с требованиями, указанными в подразделах настоящего Технического задания.

-	Клиентская часть должна включать:
-	платформу для доступа с персонального компьютера (далее – ПК) и мобильных устройств;
-	платформа должна поддерживать работу в наиболее популярных браузерах (работа на ПК): 
-	Yandex Browser версии 22 и выше;
-	 Google Chrome версии 118 и выше;
-	Mozilla Firefox версии 118 и выше;
-	Opera версии 104 и выше;
-	Apple Safari версии 16 и выше;
-	Microsoft Edge версии 118 и выше.

Клиентская часть должна обладать интуитивно понятным интерфейсом.
Требования к пользователям Системы:
-	Для пользования платформой от сотрудников организации атомной отрасли не должно требоваться специальных технических навыков, знания технологий или программных продуктов, за исключением общих навыков работы с ПК и смартфоном и/или планшетом.
Требования к сохранности информации:
В Системе должен быть предусмотрен механизм резервного копирования
структуры и содержимого базы данных. Процедура резервного копирования
производится автоматически, раз в 24 часа.
Требования к разграничению доступа:
Для любого раздела, размещенного в Системе, должна быть реализована возможность настройки доступа к разделу для определенной группы (групп) пользователей. Количество групп пользователей и количество пользователей в группе не ограничено. Администраторы Системы могут формировать произвольные группы пользователей по необходимым им критериям. Перечень сотрудников, имеющих права администратора, предоставляются Заказчиком Исполнителю в рабочем порядке, но не позднее 3 (трех) рабочих дней с даты подписания договора. Группы пользователей, ограничения и запреты на доступ к контенту определяются и добавляются из панели администрирования. Помимо групп пользователей, доступ и возможности всех пользователей в Системе должны определяться их категорией доступа. Должно быть доступно как минимум 3 (три) категории доступа:
-  обычные пользователи (участники)
- организаторы;
- администраторы.
Каждому из пользователей присваивается одна из категорий доступа. Обычные пользователи имеют доступ только к той части контента, которая доступна для тех групп пользователей, к которым относится данный пользователь. Организаторы имеют доступ к той части контента, которая доступна для их групп пользователей, а также им должен предоставляться отдельный интерфейс для отправки уведомлений, а также особые права по модерированию сообщений в новостных лентах и общих чатах. Администраторы имеют доступ к административной части. Администраторы должны иметь возможность:
-	добавлять пользователей с указанием их категории доступа;
-	добавлять и удалять разделы в платформе;
-	настраивать доступность разделов для различных категорий участников;
-	администрировать материалы события;
-	управлять контентом события для всех категорий участников;
-	формировать отчетность. Пользователь Системы может относиться одновременно к двум категориям доступа и входить в несколько групп пользователей.
Требования к функциям и задачам, выполняемым Системой. 
Структура платформы и внешний вид.
Главное меню клиентской части Системы должно быть представлено в виде набора разделов, навигация между которыми осуществляется через панель ячеек с внутренними страницами, раскрывающими подробности по конкретному разделу. Система должна позволять администраторам заводить в рамках одной платформы несколько событий с индивидуальными настройками, используемым функционалом, контентом и списком пользователей в каждом событии. Система должна позволять администраторам гибко настраивать набор модулей и доступных разделов события под решаемую задачу:
-	 гибко настраивать главное меню: добавлять новые разделы,редактировать и скрывать разделы;
-	менять названия разделов, выбирать иконки разделов из доступного набора, настраивать порядок следования разделов;
-	настраивать доступ к разделам по группам пользователей (категориям участников).
-	Система должна позволять настраивать внешний вид платформы в соответствии с брендбуком Заказчика, а именно:
-	использование фирменных цветов;
-	использование логотипа;
-	возможность создания «white label» интерфейса Системы.
Навигация
Пользовательский интерфейс платформы должен обеспечивать наглядное, интуитивно понятное представление структуры, размещенной информации, быстрый и логичный переход к разделам и страницам. Навигационные элементы должны обеспечивать однозначное понимание пользователем их смысла: ссылки на страницы должны быть снабжены заголовками, условные обозначения соответствовать общепринятым.
Система должна обеспечивать навигацию по всем ресурсам, доступным пользователю, и отображать соответствующую информацию.

Техническое сопровождение Системы на период проведения мероприятия (12.05.2025 г по 16.12.2025 г.) в рабочее время. Удаление данных и завершение доступа в срок не позднее 20.12.2025 г.

Подраздел 2.2 Описание требований к функциям и задачам, выполняемым Системой
Авторизация
Модуль для авторизации пользователя в своем профиле. Для каждого события
должен быть предусмотрен свой набор профилей пользователей. Возможности
авторизации должны включать:
- авторизацию через логин-пароль, где в качестве логина может выступать
номер телефона, email или автологин;
- возможность настроить SSO авторизацию через сторонний сервис,
поддерживающий протоколы OAuth 2.0, OpenID Connect;
- используемый способ авторизации должен выбираться в панели
администрирования данного события.
Курсы
Модуль для размещения в системе обучающих курсов. Каждый курс должен состоять из отдельных этапов, которые нужно пройти для завершения курса. Контент и требования для выполнения каждого этапа должны настраиваться индивидуально. В том числе, модуль должен включать:
●	возможность произвольного и последовательного прохождения этапов;
●	возможность ограничить доступ к некоторым этапам;
●	возможность использовать электронные курсы формата SCORM (SCORM v1.2, SCORM v2004).
База знаний
Модуль для размещения в системе контента различного типа - текста, видео, ссылок, изображений, аттачментов (PDF, презентации PowerPoint (PPTX) и др.). Модуль должен поддерживать разделение доступа к контенту для разных групп пользователей, а также создание иерархии разделов с помощью системы вложенности. Функционал должен предоставлять следующие возможности:
●	возможность создания неограниченного количества разделов;
●	сегментация по группам обучающихся, отделам, подразделениям;
●	совмещение в одном разделе базы данных материалов разного формата: лонгриды, видео, аттачменты и т.д.
●	выведение в главное меню отдельных разделов.
Кабинет руководителя
Модуль для работы кураторов и руководителей обучения. Должен предоставлять кураторам и руководителям обучения возможность следить за активностью обучающихся и результатами их обучения. Функционал должен предоставлять следующие возможности:
●	возможность отслеживать показатели активности как по нескольким группам обучения, так и по всем группам;
●	отображение списков обучающихся и кураторов для каждой группы;
●	возможность отследить прогресс обучения как по каждому обучающемуся, так и по каждому курсу;
●	возможность отправить отчет по обучению на электронную почту куратора обучения или руководителя.
6.	Общий чат
Модуль для организации общения пользователей на платформе. У всех пользователей должна быть возможность отправлять сообщения в общий чат. Сообщения должны отображаться в формате диалогов чата. Функционал должен предоставлять следующие возможности:
●	возможность отправить текст или изображение;
●	возможность ответить на сообщение другого пользователя;
●	возможность для пользователей отметить выбранное сообщение (поставить «лайк»);
●	возможность пользователям удалять свои сообщения, а организаторам обучения удалять любые сообщения.
Чаты по группам обучения
Модуль для размещения в системе чатов для общения в рамках своих групп обучения. Каждый чат по группам обучения должен быть виден только тем пользователям, которые входят в данную группу обучения. Сообщения должны отображаться в формате диалогов чата.
Геймификация
Модуль для размещения в платформе системы наград и личного рейтинга пользователей. В рамках этого модуля должно реализовываться начисление баллов за выполнение активностей в системе, а также формирование рейтинга пользователей. Функционал должен предоставлять следующие возможности:
●	возможность ручного начисления баллов пользователям с указанием причины начисления;
●	ряд автоматических начислений за действия в системе. Для каждого автоматического начисления должно быть можно настроить название начисления, количество начисляемых баллов, сколько раз баллы могут быть начислены по этому виду начисления;
●	помимо личного рейтинга, должна быть возможность формирования рейтинга по выбранным группам пользователей.
Бронирования
Модуль для размещения в системе функционала записи пользователей на определенные события. В панели администрирования должны собираться списки записавшихся. Функционал должен предоставлять следующие возможности:
●	возможность включения и выключения списка ожидания;
●	возможность скрытия списка записавшихся пользователей;
●	возможность разграничить записи по типам и ограничить количество записей на события одного типа;
●	возможность отобразить все доступные пользователю записи в одном разделе.
Опросы 
Модуль для размещения в системе опросов для сбора обратной связи и проведения голосования. Включает как общие опросы по всем пользователям, так и опросы, проводимые в рамках определенного обучающего мероприятия или какой-либо активности. Функционал должен предоставлять следующие возможности:
●	возможность добавить в опрос произвольное количество вопросов;
●	возможность объединять несколько вопросов в группу вопросов;
●	оценку по шкале от 1 до 10;
●	возможность сделать опрос активным по расписанию;
●	отправка уведомления пользователя о необходимости пройти опрос.
Тесты
Модуль для размещения в системе тестов для проверки знаний пользователей. Должно быть предусмотрено как самостоятельное использование тестов, так и в рамках обучающих мероприятий (курсов, траекторий обучения). Кроме того, функционал должен предоставлять следующие возможности:
●	настройку индикации правильных/неправильных ответов пользователей;
●	возможность добавления тестов в главное меню;
●	«защиту от списывания» (перемешивание вопросов и вариантов ответов на вопрос).
Траектории обучения
Модуль для размещения в системе комплексных обучающих программ. Траектории обучения должны позволять объединять в единую систему несколько курсов. Контент и доступ к каждому курсу должны настраиваться индивидуально. Функционал должен предоставлять следующие возможности:
●	автоматическую отправку уведомлений при назначении траектории обучения;
●	открытие шагов по расписанию;
●	отображение прогресса прохождения траектории обучения;
●	возможность ограничить доступ к курсам в рамках траектории обучения.
Трансляция 
Модуль для размещения в системе обучающих вебинаров и семинаров. Настройки каждой трансляции должны задаваться раздельно. Функционал должен предоставлять следующие возможности:
●	возможность размещения как онлайн трансляций, так и записей трансляций;
●	возможность ограничения доступа к каждой трансляции для определенных категорий пользователей;
●	возможность проигрывания видеопотоков из сервисов Youtube, Vimeo, Vk.
Задания
Модуль для размещения в системе контента в формате заданий. Каждое задание должно состоять из набора полей, которые необходимо заполнить до отправки задания на проверку. Проверка ответов пользователей должна осуществляться в панели администрирования. Функционал должен предоставлять следующие возможности:
●	возможность добавлять в поля текст, ссылки (с валидацией корректности указанных ссылок) и файлы;
●	возможность прикреплять файлы к описанию задания (например, шаблон проекта);
●	возможность возврата задания на доработку произвольное количество раз;
●	возможность прикрепления файлов к комментариям проверяющего задание преподавателя;
●	возможность настройки условий прохождения задания - сразу после отправки или только после проверки преподавателем.
Лента
Модуль для размещения в системе новостей компании, новостей по процессу и программам обучения. Функционал должен предоставлять следующие возможности:
●	возможность добавить в новость текстовую информацию и изображение;
●	возможность добавить новость из панели администрирования для пользователей с правами организаторов обучения;
●	возможность для пользователей отметить выбранную новость (поставить «лайк»);
●	возможность продублировать отправленное сообщение push-уведомлением.
Материалы и презентации
Модуль для размещения в системе информационных разделов произвольного содержания. Каждый такой раздел должен настраиваться отдельно. Функционал должен предоставлять следующие возможности:
●	возможность добавления любого количества информационных разделов и выведение их в главное меню;
●	возможность использования в тексте ссылок, изображений и табуляции;
●	возможность управления доступом к каждому разделу.
Отчетность и статистика
Модуль для сбора статистики по действиям пользователе в системе. Должен включать в себя как отображение статистики в панели администрирования, так и возможность выгрузки отчетов. В том числе, модуль должен включать:
●	выгрузку Excel-отчетов по результатам обучения (курсам, траекториям обучения, тестам т.д.);
●	отображение основной статистики по системе в панели администрирования;
●	возможность расширения набора отчетов через построение кастомных запросов в формате SQL.
Участники и кураторы
Отображение профилей участников программы обучения. 
Профиль будет включать в себя:
●	ФИО;
●	Аватарку пользователя;
Если у пользователя не загружена аватарка, будет показываться плейсхолдер.
●	Должность (если указана);
●	Телефон 
●	Почта
