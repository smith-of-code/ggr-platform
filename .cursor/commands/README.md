# Cursor-команды проекта

Команды вызываются в чате Cursor через `/command-name`. После вызова нужно передать параметры текстом.

---

## /feature — Планирование фичи

Создаёт (или дополняет) спецификацию, план, задачи и трекер прогресса для новой фичи.

```
slug: my-feature
intent: Краткое описание цели (1-2 предложения)
constraints: (опц.)
out_of_scope: (опц.)
```

Результат: `/spec/features/<slug>/` — `spec.md`, `plan.md`, `tasks.md`, `progress.md`.
Если фича уже существует — прогресс сохраняется, новые задачи добавляются.

---

## /feature-work — Выполнение задач

Берёт следующую задачу из `progress.md` и выполняет шаг(и).

```
slug: my-feature
mode: normal | fast   (по умолчанию normal)
max_steps: 99         (только для fast)
```

- `normal` — 1 шаг за вызов
- `fast` — до `max_steps` шагов подряд

Автоматически: линтер, тесты, обновление `progress.md`.

---

## /feature-fix — Исправление бага

Фиксирует баг в спеке, создаёт задачи на исправление и выполняет их.

```
slug: my-feature
issue: Ожидалось X, получилось Y. Шаги: ...
mode: normal | fast   (по умолчанию normal)
max_steps: 3          (только для fast)
```

Добавляет регрессионный тест, если баг связан с логикой.

---

## /spec-sync — Синхронизация спеки с кодом

Приводит `/spec` в соответствие с актуальным кодом (код — источник истины). Если `/spec` не существует — создаёт с нуля.

```
scope: my-feature | global
dry_run: true | false   (по умолчанию false)
```

- `global` — проверяет все модули, модели, маршруты
- `dry_run: true` — только показывает расхождения, не меняет файлы

---

## /repo-analyze — Анализ репозитория

Отвечает на вопросы о коде и архитектуре. Ничего не меняет (read-only).

```
questions: Какие модели есть? Как работает авторизация?
scope: my-feature | module | путь   (опц.)
```

Ответы со ссылками на файлы и уровнем уверенности (high / medium / low).

---

## /refactor — Рефакторинг

Улучшает структуру кода без изменения поведения.

```
target: app/Services/MyService.php | директория | класс
goal: extract service | reduce duplication | apply SRP
max_files: 5   (опц.)
```

Прогоняет тесты до и после, откатывает при поломке.

---

## /e2e-cases — E2E тесткейсы

Генерирует человекочитаемые тесткейсы (словами, без кода). Сохраняет в `/spec/e2e/<scope>.md`.

```
scope: lms-courses | lms-auth | global
focus: CRUD курсов              (опц.)
role: участник | админ | гость  (опц.)
```

Каждый кейс: роль, предусловия, шаги пользователя, ожидаемый результат.
При повторном запуске — дополняет файл, не дублирует существующие кейсы.

---

## Типичный workflow

```
/feature        slug=my-feature intent="Добавить каталог курсов"
                    ↓
/feature-work   slug=my-feature                    (по 1 шагу)
/feature-work   slug=my-feature mode=fast          (пакетно)
                    ↓
/feature-fix    slug=my-feature issue="..."        (если баг)
                    ↓
/e2e-cases      scope=my-feature                   (тесткейсы)
                    ↓
/spec-sync      scope=my-feature                   (синхронизация спеки)
```

Для разведки в любой момент: `/repo-analyze questions="..."`.
