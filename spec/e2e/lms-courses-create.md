# E2E тесткейсы: Создание и редактирование курсов (LMS Admin)

**Scope**: Админка LMS → CRUD курсов, модули, этапы, копирование из других курсов
**Маршруты**: `/lms-admin/{event}/courses/*`, `/lms-admin/{event}/search-modules`, `/lms-admin/{event}/search-stages`

---

## Список курсов

### TC-001: Просмотр списка курсов
**Роль:** админ
**Предусловия:** Существует мероприятие с 2+ курсами (один активен, другой скрыт)

**Шаги:**
1. Перейти на страницу `/lms-admin/{event}/courses`

**Ожидаемый результат:**
- Отображается таблица курсов с колонками: Название, Этапов, Записей, Статус, Действия
- Активный курс имеет бейдж «Активен» (зелёный)
- Скрытый курс имеет бейдж «Скрыт» (серый)
- У каждого курса есть кнопки редактирования (карандаш) и удаления (корзина)
- В шапке страницы есть кнопка «Создать курс»

---

### TC-002: Пустой список курсов
**Роль:** админ
**Предусловия:** Мероприятие существует, курсов нет

**Шаги:**
1. Перейти на страницу `/lms-admin/{event}/courses`

**Ожидаемый результат:**
- Отображается сообщение «Курсов пока нет»
- Кнопка «Создать курс» доступна в шапке

---

## Создание курса — основная информация

### TC-003: Открытие формы создания курса
**Роль:** админ
**Предусловия:** Существует мероприятие

**Шаги:**
1. Перейти на страницу списка курсов `/lms-admin/{event}/courses`
2. Нажать кнопку «Создать курс»

**Ожидаемый результат:**
- Открывается страница с заголовком «Новый курс»
- Есть ссылка «Назад к курсам»
- Форма содержит поля: Название, Slug, Описание (WYSIWYG-редактор), Изображение курса
- Чекбоксы: «Последовательное прохождение» (включён по умолчанию), «Активен» (включён по умолчанию), «Требуется одобрение для записи» (выключен по умолчанию)
- Блок «Модули» пустой с подсказкой «Модулей пока нет. Нажмите «Добавить модуль», чтобы структурировать курс.»
- Блок «Этапы без модуля» содержит один пустой этап
- Кнопки «Сохранить» и «Отмена»

---

### TC-004: Создание курса с минимальными данными (happy path)
**Роль:** админ
**Предусловия:** Существует мероприятие

**Шаги:**
1. Открыть форму создания курса
2. Ввести в поле «Название» значение «Базовый курс»
3. Нажать кнопку «Сохранить»

**Ожидаемый результат:**
- Перенаправление на страницу списка курсов
- Появляется flash-сообщение «Курс создан»
- В списке отображается курс «Базовый курс» со статусом «Активен»
- Slug автоматически сгенерирован как «bazovyj-kurs» (или аналог)

---

### TC-005: Создание курса со всеми полями
**Роль:** админ
**Предусловия:** Существует мероприятие

**Шаги:**
1. Открыть форму создания курса
2. Ввести название: «Продвинутый курс»
3. Ввести slug: «advanced-course»
4. Ввести описание в WYSIWYG-редакторе: «Описание курса»
5. Снять чекбокс «Последовательное прохождение»
6. Установить чекбокс «Требуется одобрение для записи»
7. Нажать кнопку «Сохранить»

**Ожидаемый результат:**
- Перенаправление на список курсов
- Появляется flash-сообщение «Курс создан»
- Курс «Продвинутый курс» отображается в списке

### TC-010: Добавление нескольких модулей и изменение порядка

## Копирование модулей и этапов из других курсов

### TC-023: Поиск и копирование модуля из другого курса
**Роль:** админ
**Предусловия:** В том же мероприятии существует курс «Курс-источник» с модулем «Модуль-шаблон» (содержит 3 этапа)

**Шаги:**
1. Открыть форму создания нового курса
2. Нажать «Добавить модуль»
3. Нажать ссылку «Найти модуль из другого курса» (видна, пока название пустое)
4. В открывшемся попапе ввести «Модуль-шаблон»
5. Дождаться появления результатов (debounce 300ms)
6. Нажать на результат «Модуль-шаблон» (Курс: Курс-источник · 3 этап(ов))
7. В появившемся подтверждении нажать кнопку «Копировать»

**Ожидаемый результат:**
- Попап закрывается
- Поле «Название модуля» заполняется значением «Модуль-шаблон»
- Поле «Описание» заполняется описанием из источника
- Даты «Открытие» / «Закрытие» копируются (если были)
- Внутри модуля появляются 3 скопированных этапа с названиями и типами из источника
- Ссылка «Найти модуль из другого курса» скрывается (т.к. название заполнено)
- Скопированные данные можно далее редактировать
