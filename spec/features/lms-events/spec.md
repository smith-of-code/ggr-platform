# Фича: LMS — События

**Статус**: implemented

## Описание

Событие (LmsEvent) — корневая сущность LMS-платформы. Каждое событие изолирует свой набор участников, контент (курсы, тесты, задания, траектории, видео, БЗ, материалы, геймификация, группы) и настройки меню. Внутри одной платформы можно создать несколько событий с независимыми конфигурациями.

## Связанные сущности

### Модели
- `App\Models\Lms\LmsEvent`

### Контроллеры
- `App\Http\Controllers\Lms\Admin\EventController` — CRUD событий

### Страницы
- `Pages/Lms/Admin/Events/Index.vue` — список событий
- `Pages/Lms/Admin/Events/Create.vue` — создание события
- `Pages/Lms/Admin/Events/Edit.vue` — редактирование события

## Поля модели LmsEvent

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | string | Название события |
| `slug` | string | URL-слаг (автогенерация из title) |
| `description` | text | Описание события |
| `auth_method` | string | Способ авторизации (сейчас только `email`) |
| `is_active` | boolean | Активность события |
| `menu_config` | JSON | Настройка видимости разделов меню |

## Настройка меню (menu_config)

Администратор может включать/выключать разделы бокового меню для каждого события отдельно. Конфигурация хранится в JSON-поле `menu_config`.

### Доступные разделы

| Ключ | Раздел | По умолчанию |
|------|--------|:------------:|
| `courses` | Курсы | включён |
| `trajectories` | Траектории | включён |
| `tests` | Тестирование | включён |
| `assignments` | Задания | включён |
| `leaderboard` | Рейтинг | включён |
| `videos` | Видеоматериалы | включён |
| `kb` | База знаний | включён |
| `materials` | Материалы | включён |

Разделы «Главная», «Кабинет лидера» и «Админ-панель LMS» не настраиваются — они зависят от роли пользователя.

### Как работает

1. При создании/редактировании события в формах отображаются чекбоксы разделов
2. `LmsEvent::getMenuConfigAttribute()` мержит сохранённую конфигурацию с дефолтами
3. Все participant-facing контроллеры передают `menu_config` в Inertia-ответе
4. `LmsLayout.vue` фильтрует `sidebarItems` по `menu_config` события

## Изоляция контента по событию

Все LMS-сущности привязаны к событию через FK `lms_event_id`:

- `lms_profiles` — участники
- `lms_courses` — курсы
- `lms_tests` — тесты
- `lms_assignments` — задания
- `lms_trajectories` — траектории
- `lms_videos` — видео
- `lms_kb_sections` — база знаний
- `lms_material_sections` — материалы
- `lms_groups` — группы
- `lms_gamification_rules` — правила геймификации
- `lms_gamification_points` — баллы
- `lms_roles` — роли

Cascade delete: удаление события удаляет весь связанный контент.

## Авторизация

Текущая реализация поддерживает только email + пароль. Поле `auth_method` сохраняется в БД, но не влияет на логику авторизации в `AuthController`. SSO (OAuth 2.0, OpenID Connect) не реализован — UI выбора SSO удалён из форм.

## Ключевые workflow

- Создание события → автогенерация slug → настройка меню → активация
- Каждое событие получает свой URL: `/lms/{slug}/`
- Участники привязываются к событию через `LmsProfile`
- Контент создаётся в рамках события через LMS-админку (`/lms-admin/{event}/...`)
