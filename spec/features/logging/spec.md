# Логирование в базу данных

## Цель

Перенести логирование ключевых событий (активность пользователей, исключения) в базу данных для централизованного хранения и анализа.

## В скоупе

- Миграции для таблиц `activity_logs` (HTTP-запросы аутентифицированных пользователей) и `exception_logs` (все исключения приложения)
- Eloquent-модели `ActivityLog`, `ExceptionLog`
- Сервис `ActivityLogService` для записи логов
- Middleware `LogUserActivity` для автоматической записи HTTP-активности
- Кастомизация exception handler в `bootstrap/app.php` для записи исключений в БД
- Конфигурация `config/activity-logging.php`: вкл/выкл логирование, вкл/выкл логирование 422 ошибок, исключённые маршруты, retention (дни)
- Artisan-команда для очистки устаревших записей

## Вне скоупа

- UI для просмотра логов (будет отдельной фичей)
- Интеграция с внешними сервисами (Sentry, ELK и т.д.)
- Замена файлового логирования (остаётся параллельно)
- Логирование CLI-команд (только HTTP)

## Ограничения

- Запись логов не должна ломать основной flow при ошибке (silent fail)
- Логирование 422 по умолчанию выключено (включается через конфиг)
- Middleware добавляется в web-группу через `bootstrap/app.php`
- Все команды выполняются только в Docker (`source docker/.env.local && docker exec ${APP_NAME}_fpm ...`)
