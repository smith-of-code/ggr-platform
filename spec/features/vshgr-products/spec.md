# Типы продуктов ВШГР

## Статус: Реализован

## Цель

Три типа образовательных продуктов с разными наборами полей в админке и разным визуалом на публичной части.

---

## Типы продуктов

### 1. `education` — Продукты образования

Основной тип (~8 штук). Пример: `rosatom-travel.ru/vshgr/osnovy-turisticheskogo-menedzhmenta`.

**Секции (блочная структура, каждая включается/выключается чекбоксом в админке):**


| #   | Секция (slug)        | Содержимое                        | Тип контента                                      |
| --- | -------------------- | --------------------------------- | ------------------------------------------------- |
| 1   | `description_goal`   | Описание и цель                   | RichText                                          |
| 2   | `results`            | Результаты работы                 | RichText                                          |
| 3   | `streams_formats`    | Потоки / форматы                  | RichText                                          |
| 4   | `target_audience`    | Целевая аудитория                 | RichText                                          |
| 5   | `directions`         | Направления                       | RichText                                          |
| 6   | `what_we_offer`      | Что мы предлагаем                 | RichText                                          |
| 7   | `selection_criteria` | Критерии участия / отбора         | RichText                                          |
| 8   | `experts`            | Эксперты                          | JSON-массив: `[{name, position, photo, bio}]`     |
| 9   | `best_cases`         | Лучшие кейсы / прошлый опыт       | RichText                                          |
| 10  | `regulation`         | Положение                         | PDF-файл (загрузка через медиа-систему)           |
| 11  | `application_form`   | Подача заявки                     | Встроенная форма (useForm → `applications.store`) |
| 12  | `training_request`   | Заявка на обучение в регионе / КП | RichText + форма или ссылка                       |


**Форма заявки** — та же что есть сейчас (`type: program_info`), с валидацией email (rfc,dns) и маской телефона.

### 2. `partner` — Партнёрские программы

Одна карточка в каталоге → отдельная страница.

**Секции:**


| #   | Секция (slug)              | Содержимое                    | Тип контента |
| --- | -------------------------- | ----------------------------- | ------------ |
| 1   | `description_goal`         | Описание партнёрских программ | RichText     |
| 2   | `participation_conditions` | Условия участия               | RichText     |


### 3. `international` — Международный контур

Одна карточка → страница с разделением на страны (Абхазия, Беларусь) внутри одной страницы.

**Секции (MVP — только `description_goal`, остальные на уточнении):**


| #   | Секция (slug)      | Содержимое      | Тип контента |
| --- | ------------------ | --------------- | ------------ |
| 1   | `description_goal` | Описание и цель | RichText     |


**Планируемые секции (заблокированы, ожидают уточнения):**

- `streams_formats`, `target_audience`, `directions`, `what_we_offer`, `selection_criteria`, `experts`, `regulation`, `application_form`, `training_request`

**Страновая структура:**
Поле `countries` (JSON) — массив объектов `[{name, slug, description, content}]`.
На публичной странице — визуальное разделение на блоки по странам (tabs или anchor-навигация).

---

## In-scope

- Добавление поля `type` (`education` | `partner` | `international`) в модель `EducationProduct`
- Добавление JSON-поля `sections` для хранения блочных данных
- Добавление JSON-поля `countries` для международного типа
- Добавление поля `regulation_file` для PDF-файла положения
- Выбор типа при создании нового продукта (кнопка «Новый продукт» → выбор типа)
- Динамическая админ-форма с полями, зависящими от типа
- Чекбоксы для включения/выключения секций (тип `education`)
- Три разных шаблона публичной страницы `Education/Show.vue` (по типу)
- Валидация email/phone в форме заявки (уже реализована — REV-001)
- Загрузка PDF для положения через медиа-систему

## Out-of-scope

- LMS-курсы (уже реализованы отдельно)
- Блок социальных сетей (уже реализован — REV-002)
- Расширенные секции для типа `international` (ожидают уточнения)
- Эксперты как отдельная сущность / CRUD (пока JSON в sections)

## Constraints

- Обратная совместимость: существующие продукты (`type = null`) автоматически становятся `education`
- Миграция: `type` default `education`, `sections` / `countries` nullable JSON
- Максимум 5 файлов за шаг, не более 150 строк diff за шаг
- Все команды через Docker (`source docker/.env.local && docker exec ${APP_NAME}_fpm ...`)
- Переиспользование: `RichTextEditor`, `ImageUploadCrop`, `RInput`, `RCheckbox`, `RCard`, `RButton`, `ContentPreview`

## Открытые вопросы

1. **Эксперты** — формат отображения на публичной части (карточки с фото? список? карусель?). Пока реализуем карточки с фото, именем, должностью, кратким био.
2. **training_request** — формат блока «Заявка на обучение в регионе»: форма или ссылка на скачивание КП? Пока реализуем как RichText + кнопку-ссылку на внешний URL или скачивание файла.
3. **Международный контур** — точный состав блоков для Абхазии и Беларуси (на уточнении у заказчика).
