# План: S3 Integrate

## Milestones

1. **Конфигурация** — добавить env-переменные для S3, обновить `config/filesystems.php`, добавить переменные в `.env.prod` / `.env.example`
2. **Абстракция диска** — создать хелпер/метод для определения целевого диска загрузок, чтобы не дублировать логику
3. **Рефакторинг контроллеров** — заменить хардкод `'public'` на конфигурируемый диск во всех контроллерах загрузки
4. **Рефакторинг URL** — заменить `'/storage/' . $path` на `Storage::url($path)` во всех контроллерах
5. **Отдача файлов** — обеспечить корректную отдачу файлов на фронте (URL из S3 vs локальный `/storage/`)
6. **Проверка** — убедиться что локально всё работает как прежде

## Архитектурные решения

- **Диск загрузок**: используем `config('filesystems.default')` или отдельную переменную `UPLOAD_DISK`. Решение: используем `UPLOAD_DISK` (default: `public`) — позволяет отвязать диск загрузок от системного default.
- **URL**: `Storage::disk($disk)->url($path)` — единообразно для S3 и local.
- **SCORM**: оставляем в `public_path()` — это статический контент, извлекается как HTML/JS, не подходит для S3-хранения (нужна раздача из CDN/nginx).

## Verification

Все проверки выполняются по Command Execution Pattern из spec-continuation (Docker exec).
