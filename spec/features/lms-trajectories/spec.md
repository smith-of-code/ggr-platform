## Ключевые workflow

- Траектория содержит шаги (steps), каждый ссылается на курс
- Шаги могут быть заблокированы (is_locked) и открываться по расписанию (opens_at)
- Позиционирование шагов (position) определяет порядок
- Запись на траекторию: enrolled → in_progress → completed

### Блоки траектории (static / task) и программы

- У блока в БД поле `visible_course_ids` (JSON, nullable): **`null` или пустой список** — блок виден **всем** участникам события (поведение по умолчанию).
- Если указаны id курсов (`lms_courses` этого же события), блок в ленте участника показывается **только** при записи пользователя **хотя бы на один** из этих курсов.
- Участник **без ни одной записи на курс** в событии: лента траектории **не строится**; на странице показывается текст о необходимости записаться на программу и ссылка на каталог курсов (`lms.courses.index`).
- Карточки программ и грантов в ленте по-прежнему строятся из фактических записей пользователя (после появления записи на курс лента доступна и включает соответствующие элементы).
