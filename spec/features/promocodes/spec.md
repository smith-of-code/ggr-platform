# Промокоды

## Цель

Реализовать систему промокодов со скидкой на туры: CRUD в админке, применение промокода при подаче заявки с мгновенным отображением цены со скидкой.

## В скоупе

- Модель `Promocode` (код, процент скидки, срок действия, привязка к сущности — по умолчанию `Tour`, is_active)
- Полиморфная привязка промокода к сущности (`promocodeable_type` / `promocodeable_id`); в UI привязка не показывается — по умолчанию к туру
- Генерация кода: из названия (транслитерация/slug), случайные символы или произвольный текст латиницей
- Админка: CRUD промокодов (список, создание, редактирование, удаление, toggle active)
- Публичная страница тура (`Tours/Show.vue`): кнопка «У меня есть промокод», поле ввода, валидация через API
- Мгновенное отображение цены со скидкой (price_from в шапке, цена в карточках заездов, sidebar)
- Сохранение `promocode_id` в заявке (`Application`) при отправке формы
- Валидация промокода: существует, активен, не истёк, привязан к данному туру (или без привязки)

## Вне скоупа

- Лимит использований промокода
- Аналитика / статистика промокодов
- Промокоды для других сущностей (кроме туров) — архитектура полиморфная, но UI только для туров
- Уведомления об истечении промокода
- Массовая генерация промокодов

## Ограничения

- Промокод — процент скидки (не фиксированная сумма)
- Срок действия обязателен (дата начала + дата окончания)
- Код уникален в системе
- Скидка применяется только к отображению цены на фронте + фиксируется в заявке; реальная оплата/биллинг отсутствует
- Привязка к сущности по умолчанию — Tour; в UI выбор типа сущности не показываем
