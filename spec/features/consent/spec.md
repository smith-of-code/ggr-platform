# Consent — логирование согласий на обработку персональных данных

## Цель

Реализовать централизованное логирование факта согласия пользователя на обработку персональных данных с фиксацией IP, User-Agent, страницы, версии политики и идентификаторов пользователя.

## В scope

- Миграция таблицы `consents` (на основе предоставленной схемы)
- Модель `Consent`
- Сервис `ConsentService` для записи событий согласия
- Интеграция логирования в формы регистрации портала (`Auth/Register`)
- Интеграция логирования в формы LMS (`Lms/Auth/Register`, `Invite`, `Activate`)
- Интеграция логирования в формы заявок (`ApplicationController`)
- Интеграция логирования в контактную форму (`HomeController::contactSubmit`)
- Добавление чекбокса согласия в формы, где он отсутствует (заявки, контактная форма)

## Вне scope

- Cookie consent banner (GDPR cookies)
- Механизм отзыва согласия
- GDPR data export / right to be forgotten
- Админ-панель просмотра логов согласий (отдельная фича)
- Интеграция с LMS public forms (FormRenderer) — динамические формы имеют свою схему полей

## Ограничения

- Все команды выполняются только в Docker
- Запись согласия — синхронная (без очередей), т.к. это критичное юридическое действие
- `policy_version` по умолчанию `'1.0'`, обновляется вручную при изменении политики
- `event_type` — строковый идентификатор формы/действия (e.g. `registration`, `lms_registration`, `application`, `contact_form`)
- Таблица только на запись (append-only), записи не удаляются и не обновляются
