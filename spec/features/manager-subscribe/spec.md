# manager-subscribe — Управление подписчиками рассылки в админке

## Цель

Добавить в админ-панель полноценный CRUD для управления подписчиками блога (`blog_subscribers`): просмотр списка, добавление, приостановка/активация, удаление.

## В скоупе

- Страница списка подписчиков с пагинацией и поиском по email
- Кнопка добавления нового подписчика (ввод email)
- Toggle активности подписчика (`is_active`)
- Удаление подписчика с подтверждением
- Отображение статуса (активен / приостановлен) бейджем
- Статистика: общее количество / активных / приостановленных

## Вне скоупа

- Отправка рассылок из админки
- Импорт/экспорт CSV
- Редактирование email подписчика (только добавление/удаление)
- Интеграция с внешними сервисами рассылок

## Ограничения

- Используется существующая модель `BlogSubscriber` (таблица `blog_subscribers`, поля: `email`, `is_active`, `token`)
- Админка — Laravel + Inertia.js + Vue 3, layout `AdminLayout`
- Маршруты внутри `Route::prefix('admin')->name('admin.')` группы
- Контроллер в `App\Http\Controllers\Admin\`
- Vue-страницы в `resources/js/Pages/Admin/Subscribers/`
- Паттерн CRUD аналогичен `CityController` / `Cities/Index.vue`
- Новые миграции НЕ нужны — таблица уже содержит все необходимые поля
