# Расширение фактов о городе: ссылка + описание с форматированием

## Цель

Превратить факты о городе из массива строк в объекты с поддержкой ссылки (URL) и rich-text описания (аналогично описанию города).

## In-scope

- Изменение структуры `facts` с `string[]` на `array<{title, url, description}>` (JSON в БД, без миграции — формат расширяется)
- Обратная совместимость: существующие строковые факты нормализуются в `{title: string, url: null, description: null}` через accessor в модели
- Обновление валидации в `Admin\CityController` (`store`, `update`): `facts.*.title`, `facts.*.url`, `facts.*.description`
- Обновление админ-формы (`Admin/Cities/Form.vue`):
  - Каждый факт — карточка: input для `title`, input для `url` (необязательно), `RichTextEditor` для `description` (необязательно)
- Обновление публичной страницы (`Cities/Show.vue`):
  - Если у факта только `title` — отображать как сейчас (чипс с текстом)
  - Если у факта есть `url` — `title` становится ссылкой
  - Если у факта есть `description` — вместо `title` в чипсе выводить `description` через `v-html` (с форматированием)

## Out-of-scope

- Отдельная таблица `city_facts` (факты остаются JSON в `cities.facts`)
- Изображения внутри факта (кроме встроенных через rich-text editor)
- Миграция БД (структура JSON расширяется без изменения колонки)

## Constraints

- Компонент `RichTextEditor` уже есть (`resources/js/Components/RichTextEditor.vue`), переиспользовать
- Обратная совместимость: старые данные `["строка"]` должны корректно читаться
- Максимум 5 файлов за шаг, до ~150 строк diff
