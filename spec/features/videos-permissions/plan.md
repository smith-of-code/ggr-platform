# videos-permissions — план

## Milestones

1. Зафиксировать правила видимости: «всем» vs набор программ; поведение списка без фильтра и с фильтром по программе; обновить валидацию и `VideoController` / `Admin\VideoController`.
2. Миграция при необходимости: только если без явного флага нельзя различить «всем намеренно» и «набор курсов пуст по ошибке» — иначе достаточно UX (чекбокс «Всем» сбрасывает `course_ids`).
3. Админский UI: блок «Кому показывать», чекбокс «Всем пользователям», список программ с галочками (переименование «Группы» → «Программы» при необходимости), взаимоисключение режимов.
4. UI участника: фильтр по программе рядом с поиском; передача `filters` из бэкенда (список программ события для селекта + выбранное значение).
5. Тесты: feature-тесты на выдачу списка и `show` с фильтром и без; регрессия для «всем пользователям».
6. Верификация сборки и тестов по правилам Docker.

## UI Components

- Админка: существующие `RCard`, `RInput`, `RButton`, `RCheckbox` (`resources/js/Pages/Lms/Admin/Videos/Form.vue`); при выборе селекта для фильтра на стороне участника — переиспользовать `@/Components/SearchSelect.vue` или тот же паттерн, что в других LMS-страницах с фильтрами (`Lms/Admin/Users/Index.vue`, `Lms/Grants/Index.vue`).
- Участник: `LmsLayout`, `RCard`, `RBadge` в `Lms/Videos/Index.vue`; новый контрол фильтра визуально согласовать с полем поиска (отступы, `rounded-xl`, focus ring как у поиска).

## Verification

По правилу **Docker-only** из `spec-continuation`: загрузить `source docker/.env.<env>`, выполнять команды через `docker exec ${APP_NAME}_fpm` (или `docker compose ... run --rm fpm`, если контейнеры не подняты). Проверить PHPUnit/Pest по затронутым тестам и при необходимости `npm run build` внутри контейнера `fpm`.
