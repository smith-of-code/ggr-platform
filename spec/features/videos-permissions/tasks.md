# videos-permissions — задачи

## Task 1: Уточнение модели видимости и контракт API

- **Goal:** Описать в коде/комментариях и валидации админки семантику: `visible_to_all` (или эквивалент) vs `group_ids`; запрет противоречивых сочетаний.
- **Scope:** `app/Http/Controllers/Lms/Admin/VideoController.php`, при необходимости миграция `lms_videos`, `app/Models/Lms/LmsVideo.php`.
- **DoD:** Админ не может сохранить неоднозначное состояние; для существующих записей определено отображение в форме.
- **Verify:** По паттерну Docker из `spec-continuation`; минимум один автотест валидации store/update.

## Task 2: Список видео участника — query без фильтра по программе

- **Goal:** Обновить выборку для индекса: «всем пользователям» и видео по программам пользователя; не ломать пагинацию и поиск по `search`.
- **Scope:** `app/Http/Controllers/Lms/VideoController.php` (`index`).
- **DoD:** Поведение согласовано со spec для нефильтрованного списка.
- **Verify:** По паттерну Docker; Pest/PHPUnit на типичные комбинации групп/«всем».

## Task 3: Список видео участника — фильтр по программе

- **Goal:** Параметр запроса (например `program_id` / `lms_group_id`) ограничивает список видео, связанных с выбранной программой; **без** требования членства участника в этой программе.
- **Scope:** `VideoController::index`, Inertia props `filters`.
- **DoD:** Участник только программы «А» видит записи «Б» при выбранном фильтре «Б» (e2e или feature-тест с фикстурами).
- **Verify:** По паттерну Docker; автотест на выдачу с фильтром.

## Task 4: Просмотр видео (`show`) и согласованность с фильтром

- **Goal:** Разрешить просмотр страницы видео, если доступ разрешён правилами spec (в т.ч. переход со списка с фильтром по программе без записи в программу).
- **Scope:** `VideoController::show`.
- **DoD:** Нет 403 для сценария из spec; сохранить защиту от чужого события и неактивных видео.
- **Verify:** По паттерну Docker; тест на `show` с фильтром/контекстом программы.

## Task 5: Админка — UI «Кому показывать»

- **Goal:** Чекбокс «Всем пользователям» + галочки программ; взаимоисключение; подписи «Программы» вместо «Группы» при необходимости.
- **Scope:** `resources/js/Pages/Lms/Admin/Videos/Form.vue`, при необходимости сериализация props с бэкенда.
- **DoD:** Сохранение создаёт/обновляет видео с ожидаемыми связями и флагом «всем».
- **Verify:** Ручная проверка в браузере или Dusk при наличии; `npm run build` в контейнере `fpm`.

## Task 6: Участник — UI фильтра по программе

- **Goal:** Рядом с поиском — выбор программы (все программы события в опциях); синхронизация с query string; сохранение пагинации.
- **Scope:** `resources/js/Pages/Lms/Videos/Index.vue`.
- **DoD:** UX согласован с существующим полем поиска; нет лишних запросов (debounce при необходимости только для поиска).
- **Verify:** `npm run build` в контейнере `fpm`.

## Task 7: Индекс админки — отображение режима видимости (опционально но желательно)

- **Goal:** В списке видео админки кратко видно «Всем» или перечисление программ.
- **Scope:** `app/Http/Controllers/Lms/Admin/VideoController.php` (`index`), `resources/js/Pages/Lms/Admin/Videos/Index.vue`.
- **DoD:** Админ видит режим без открытия формы.
- **Verify:** Ручная проверка или снимок Inertia props в тесте.

## Task 8: Регрессия и конфликт с lms-events

- **Goal:** Убедиться, что правки в `Lms/Videos/Show.vue` не ломают исправления RuTube из `lms-events`; при изменении Show — минимальный diff.
- **Scope:** `resources/js/Pages/Lms/Videos/Show.vue`, при необходимости координация с `spec/features/lms-events/progress.md`.
- **DoD:** Существующие тесты по видео/RuTube проходят; нет дублирования логики.
- **Verify:** По паттерну Docker; целевой `--filter` по тестам видео.

- **DoD:** Все задачи перенесены в Completed; Open issues пуст или задокументированы.
