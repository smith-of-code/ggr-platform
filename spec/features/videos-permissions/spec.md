# videos-permissions — «Видеоматериалы»: видимость и фильтр

## Goal

Задать, кому показывается видео при публикации админом, и дать участнику поиск плюс фильтр по программам, включая программы без записи участника.

## Соответствие формулировкам ТЗ

| Часть ТЗ | Требование | В спецификации |
|----------|------------|----------------|
| Часть 1 | При выкладке спрашивать **кому показывать**: галочки по **конкретным программам** или **«Всем пользователям»** (в т.ч. без записи ни в одну программу) | Админ-форма видео: явный режим «всем» vs набор программ; список программ — **все программы (группы) события**, см. ниже |
| Часть 2 | Поиск **и** фильтр по программам у участника | Страница «Видеоматериалы»: строка поиска + фильтр по программе |
| Часть 3 / п.3 | Фильтр показывает видео программ, **на которые участник не записан**; сценарий А → фильтр «Б» → просмотр | Выдача списка при выбранной программе **Б** и `show` не требуют членства в **Б** |

Формулировка ТЗ «список всех **записанных** программ» в части 1 в контексте **админки** трактуется как **полный перечень программ (направлений) события**, на которые участники **могут** записаться — т.е. все `LmsGroup` данного `LmsEvent`, а не «программы, где уже есть хотя бы один участник» и не «только программы текущего пользователя».

## In-scope

- Админка LMS: форма создания/редактирования видео (`lms-admin/.../videos`, маршрут `lms.admin.videos.*`) — явный выбор видимости: **«Всем пользователям»** (включая пользователей с доступом к событию LMS, **не** записанных ни в одну программу этого события) **или** галочки по **конкретным программам** из полного списка групп события (`LmsGroup`).
- Участник: страница «Видеоматериалы» (`lms/.../videos`, `lms.videos.index`) — **сохранить** строку поиска; **добавить** фильтр по программе (query-параметр + согласованная логика на бэкенде).
- Список и карточка видео: при активном фильтре «программа **Б**» участник видит видео, привязанные к **Б**, и может открыть просмотр (`show`), **без** требования записи в **Б**; ограничения — доступ к событию, `is_active`, корректный `lms_event_id`.
- Терминология в UI: **программа** = группа программы события (`LmsGroup`, поле `title` и т.д.).

## Реализация (актуально)

- Колонка `lms_videos.visible_to_all` + backfill: у видео с записями в `lms_video_access` — `false`, иначе `true`.
- Админка: режим «Всем пользователям» взаимоисключающий с выбором программ; без «всем» обязательна минимум одна программа события.
- Участник: список без фильтра — `visible_to_all` **или** пересечение программ видео с `lms_group_members`; с `?lms_group_id=` — видео, привязанные к этой программе, без требования записи; `show` — любое активное видео события для авторизованного участника события.
- Тесты: `tests/Feature/Lms/VideoPermissionsTest.php`.

## Out-of-scope

- Смена модели хранения видео вне события (всё в контексте `LmsEvent`).
- Права вне роли «участник с доступом к событию» (гостевой портал и т.п.).
- Изменение внешних плееров (RuTube/YouTube) и логики превью — см. отдельно `lms-events`.

## Constraints

- Использовать существующую связь видео–группы (`lms_video_access`, `LmsVideo::groups()`), расширяя поведение и UX; избегать дублирующих таблиц без необходимости.
- Согласовать правила: **дефолтный список** (без фильтра по программе) может по-прежнему учитывать видимость по программам и «всем»; **фильтр по конкретной программе** обязан показывать видео этой программы независимо от членства участника — по продуктовому сценарию заказчика.
- Соответствие паттернам Inertia/Vue существующих страниц `Lms/Videos/*`, `Lms/Admin/Videos/*`.
