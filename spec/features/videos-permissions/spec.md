# videos-permissions — «Видеоматериалы»: видимость и фильтр

## Goal

Задать, кому показывается видео при публикации админом, и дать участнику поиск плюс фильтр по программам, включая программы без записи участника.

## In-scope

- Админка LMS: форма создания/редактирования видео (`lms-admin/.../videos`) — явный выбор видимости: **«Всем пользователям»** (включая пользователей платформы без записи ни в одну программу события) **или** галочки по **программам** (в коде: `LmsGroup` события, список групп события).
- Участник: страница «Видеоматериалы» (`lms/.../videos`) — сохранить поиск, добавить **фильтр по программе** (query-параметр, согласованный с бэкендом).
- Бэкенд: выдача списка и просмотр отдельного видео согласованы со сценарием: при выбранной в фильтре программе **Б** участник видит и может открывать видео, помеченные программой **Б**, даже если он не записан в **Б** (при условии доступа к событию LMS и активного видео).
- Терминология в UI для админа и участника: **программа** = сущность группы программы в рамках события (`LmsGroup.title` / список групп события).

## Out-of-scope

- Смена модели хранения видео вне события (всё в контексте `LmsEvent`).
- Права вне роли «участник с доступом к событию» (гостевой портал и т.п.).
- Изменение внешних плееров (RuTube/YouTube) и логики превью — см. отдельно `lms-events`.

## Constraints

- Использовать существующую связь видео–группы (`lms_video_access`, `LmsVideo::groups()`), расширяя поведение и UX; избегать дублирующих таблиц без необходимости.
- Согласовать правила: **дефолтный список** (без фильтра по программе) может по-прежнему учитывать видимость по программам и «всем»; **фильтр по конкретной программе** обязан показывать видео этой программы независимо от членства участника — по продуктовому сценарию заказчика.
- Соответствие паттернам Inertia/Vue существующих страниц `Lms/Videos/*`, `Lms/Admin/Videos/*`.
