# videos-permissions — «Видеоматериалы»: видимость и фильтр

## Goal

Задать, кому показывается видео при публикации админом, и дать участнику поиск плюс фильтр по программам, включая программы без записи участника.

## Соответствие формулировкам ТЗ

| Часть ТЗ | Требование | В спецификации |
|----------|------------|----------------|
| Часть 1 | При выкладке спрашивать **кому показывать**: галочки по **конкретным программам** или **«Всем пользователям»** (в т.ч. без записи ни в одну программу) | Админ-форма видео: явный режим «всем» vs набор программ; список — **все учебные программы (курсы) события** (`LmsCourse`), см. ниже |
| Часть 2 | Поиск **и** фильтр по программам у участника | Страница «Видеоматериалы»: строка поиска + фильтр по программе |
| Часть 3 / п.3 | Фильтр показывает видео программ, **на которые участник не записан**; сценарий А → фильтр «Б» → просмотр | Выдача списка при выбранной программе **Б** и `show` не требуют членства в **Б** |

Формулировка ТЗ «список всех **записанных** программ» в части 1 в контексте **админки** трактуется как **полный перечень учебных программ (курсов) события** — все `LmsCourse` данного `LmsEvent` (раздел меню «Программы»), а не «только курсы, где уже есть хотя бы один участник» и не «только курсы текущего пользователя».

## In-scope

- Админка LMS: форма создания/редактирования видео (`lms-admin/.../videos`, маршрут `lms.admin.videos.*`) — явный выбор видимости: **«Всем пользователям»** (включая пользователей с доступом к событию LMS, **не** записанных ни на один курс этого события) **или** галочки по **конкретным учебным программам** из полного списка курсов события (`LmsCourse`).
- Участник: страница «Видеоматериалы» (`lms/.../videos`, `lms.videos.index`) — **сохранить** строку поиска; **добавить** фильтр по программе (query-параметр + согласованная логика на бэкенде).
- Список и карточка видео: при активном фильтре «программа **Б**» участник видит видео, привязанные к **Б**, и может открыть просмотр (`show`), **без** требования записи в **Б**; ограничения — доступ к событию, `is_active`, корректный `lms_event_id`.
- Терминология в UI: **учебная программа** в контексте видео = курс события (`LmsCourse`, поле `title` и т.д.); в меню админки пункт «Программы».

## Реализация (актуально)

- Колонка `lms_videos.visible_to_all` + backfill (исторически по pivot): у видео с привязками к курсам/ранее к группам — `false`, иначе `true`.
- Админка: режим «Всем пользователям» взаимоисключающий с выбором курсов; без «всем» обязательна минимум одна учебная программа события.
- Участник: список без фильтра — `visible_to_all` **или** пересечение курсов видео с записями пользователя в `lms_course_enrollments` по курсам этого события; с `?lms_course_id=` — видео, привязанные к этому курсу, без требования записи на курс; `show` — любое активное видео события для авторизованного участника события.
- Тесты: `tests/Feature/Lms/VideoPermissionsTest.php`.

## Out-of-scope

- Смена модели хранения видео вне события (всё в контексте `LmsEvent`).
- Права вне роли «участник с доступом к событию» (гостевой портал и т.п.).
- Изменение внешних плееров (RuTube/YouTube) и логики превью — см. отдельно `lms-events`.

## Constraints

- Связь видео–курсы: таблица `lms_video_course_access`, `LmsVideo::courses()` (many-to-many с `LmsCourse`). Ранее использовалась `lms_video_access` / группы — снята в пользу курсов.
- Согласовать правила: **дефолтный список** (без фильтра по программе) может по-прежнему учитывать видимость по программам и «всем»; **фильтр по конкретной программе** обязан показывать видео этой программы независимо от членства участника — по продуктовому сценарию заказчика.
- Соответствие паттернам Inertia/Vue существующих страниц `Lms/Videos/*`, `Lms/Admin/Videos/*`.
