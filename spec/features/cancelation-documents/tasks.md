# cancelation-documents — задачи

## Task 1: Аудит текущего потока документов

- **Goal:** Зафиксировать, как сохраняются пути файлов, какой disk, есть ли связь с `UploadedMedia`, какие Vue-страницы и маршруты задействованы.
- **Scope:** `app/Models/Lms/LmsProfileDocument.php`, `app/Http/Controllers/Lms/ProfileController.php`, `app/Http/Controllers/Lms/Admin/UserController.php`, соответствующие `routes/*.php`, `resources/js/Pages/Lms/**`.
- **DoD:** Краткая заметка в комментарии к PR или в `plan.md` (раздел рисков) при необходимости; нет противоречий с миграцией.
- **Verify:** Паттерн Docker из spec-continuation (при необходимости `php artisan route:list` / grep только в контейнере).

## Task 2: Миграция статусов и метаданных

- **Goal:** Добавить колонки для статуса документа, комментария админа, меток времени подтверждения/аннулирования; дефолты для существующих строк.
- **Scope:** `database/migrations/*_lms_profile_documents_*.php`, при необходимости константы в `LmsProfileDocument`.
- **DoD:** `migrate` проходит; старые записи получают осмысленный начальный статус (например «ожидает проверки»).
- **Verify:** Паттерн Docker; миграция up/down в dev.

## Task 3: Сервис подтверждения и аннулирования

- **Goal:** Инкапсулировать смену статуса, удаление файла из storage при аннулировании, удаление/очистку связанных записей по правилам проекта.
- **Scope:** `app/Services/` (новый или существующий), `LmsProfileDocument`, `Illuminate\Support\Facades\Storage`.
- **DoD:** Подтверждение не удаляет файл; аннулирование удаляет файл и переводит в состояние, допускающее повторную загрузку.
- **Verify:** Feature-тесты или unit на сервис; паттерн Docker.

## Task 4: Email-уведомление

- **Goal:** Письмо пользователю при аннулировании с комментарием админа (и при необходимости при подтверждении — только если продуктово нужно; минимум — аннулирование).
- **Scope:** `app/Mail/`, `resources/views/emails/` или Markdown mail, конфиг очереди при наличии.
- **DoD:** Письмо отправляется при аннулировании; текст содержит комментарий; ошибки логируются.
- **Verify:** Паттерн Docker; тест с `Mail::fake()` или ручная проверка в dev.

## Task 5: HTTP API / actions для админки

- **Goal:** Маршруты и методы контроллера для подтверждения и аннулирования документа участника; валидация; авторизация только для админов LMS.
- **Scope:** `routes/lms.php` или модуль админ-маршрутов, `app/Http/Controllers/Lms/Admin/UserController.php` (или выделенный контроллер документов), Form Request при необходимости.
- **DoD:** 403 для не-админа; 422 при пустом комментарии при аннулировании; успешный JSON/Inertia redirect по принятому стилю.
- **Verify:** Паттерн Docker; Pest/PHPUnit для авторизации и валидации.

## Task 6: Админский UI (редактирование участника)

- **Goal:** Кнопки «Подтвердить» / «Аннулировать», ввод комментария для аннулирования, отображение статуса документов.
- **Scope:** `resources/js/Pages/Lms/Admin/**` (страница пользователя/профиля), composables при необходимости.
- **DoD:** Соответствие UI kit; понятные состояния загрузки и ошибок.
- **Verify:** Паттерн Docker; `npm run build` в контейнере fpm при изменении фронта.

## Task 7: UI участника и правила загрузки

- **Goal:** После подтверждения — скрыть/отключить замену документов с пояснением; после аннулирования — снова разрешить загрузку.
- **Scope:** `resources/js/Pages/Lms/Profile/Edit.vue` (или актуальная страница документов), `ProfileController`.
- **DoD:** Нет обхода через прямой POST без проверки статуса на сервере.
- **Verify:** Паттерн Docker; ручной сценарий + тест на запрет загрузки при подтверждённом статусе.

## Task 8: Регрессия и документация спека

- **Goal:** Прогнать релевантные тесты, обновить `spec.md`/`plan.md` если фактическая реализация отличается от плана.
- **Scope:** тесты в `tests/`, файлы в `/spec/features/cancelation-documents/`.
- **DoD:** CI-эквивалент локально зелёный для затронутых тестов.
- **Verify:** Паттерн Docker (`php artisan test` с фильтром или полный набор по решению команды).

## Task 9 (опционально): Заявка «Хочу заменить документ»

- **Goal:** Кнопка у участника, комментарий, сущность заявки, админ одобряет/отклоняет; при одобрении — разблокировка загрузки для типа документа.
- **Scope:** новые миграции/модель заявки, админ + participant UI, контроллеры.
- **DoD:** Согласовано с продуктом; не ломает MVP-поток задач 1–8.
- **Verify:** Паттерн Docker; отдельные тесты на заявки.
