# cancelation-documents — план

## Milestones

1. Зафиксировать целевую модель статусов и полей в БД + обратная совместимость для уже загруженных документов.
2. Реализовать серверную логику: подтверждение, аннулирование с удалением файла из storage, транзакционная целостность.
3. Подключить уведомление по почте (шаблон, локализация при необходимости, тестовый прогон в dev).
4. Админский UI на экране редактирования участника: действия, валидация комментария, обратная связь (toast/confirm).
5. UI и правила на стороне участника: блокировка редактирования при подтверждении, сценарий повторной загрузки после аннулирования.
6. Автотесты на критичные ветки (права админа, удаление файла, смена статуса).
7. ~~Заявки на замену документа~~ — `lms_profile_document_replace_requests`, одобрение сбрасывает подтверждённый файл в состояние повторной загрузки (как аннулирование без письма).

## UI Components

- **Подтверждение действия:** `ConfirmDialog` или `Dialog` из `resources/js/components/ui/` + composable `useConfirm` при деструктивном действии (аннулирование).
- **Комментарий админа:** `Textarea` / поле формы из UI kit; при ошибках валидации — `useToast` для сообщений сервера.
- **Состояния списка документов:** `Badge` или существующие паттерны таблиц/карточек в админке LMS (выравнивание с текущей страницей пользователя в `Lms/Admin`).
- Не дублировать загрузчик файлов: использовать существующий `FileUpload` / паттерн из текущего `Profile/Edit`, если применимо.

## Verification

Проверка после каждого значимого шага: паттерн «Command Execution Pattern» из `.cursor/rules/spec-continuation.mdc` — `source docker/.env.local` и выполнение команд внутри контейнера `${APP_NAME}_fpm` (tests, build при изменении фронта). Примеры команд в спеке не дублировать.

## Аудит потока (реализация)

- Файлы документов: `config('filesystems.upload_disk')` (обычно `public`), путь вида `profile-documents/...`; связи с `UploadedMedia` нет.
- Участник: `ProfileController::uploadDocument` / `deleteDocument`, страница `resources/js/Pages/Lms/Profile/Edit.vue`.
- Админ: карточка участника `Lms/Admin/Users/Show.vue`, `UserController::show` / новые действия подтверждения и аннулирования.
- После аннулирования в БД `file_path` и `original_name` очищаются (`''`), статус `annulled`, чтобы сохранить комментарий и unique `(lms_profile_id, type)`.
