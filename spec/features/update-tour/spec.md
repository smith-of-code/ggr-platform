# update-tour — Обновление карточки тура

## Цель

Обновить отображение карточки и страницы тура: выводить города проведения чипсами, переименовать «Город старта» → «Логистические точки».

## В scope

- Города проведения (`tour.cities` из пивота `city_tour`) — выводить чипсами на карточке тура и на детальной странице
- Поле `start_city` переименовать визуально на «Логистические точки» — выводить под чипсами городов проведения
- В админке (форма и таблица): переименовать label «Город старта» → «Логистические точки»
- Формат ввода `start_city` не меняется (свободный текст, пример: «г. Волгодонск, г. Ростов-на-Дону, г. Волгоград»)
- Обновить все страницы с карточками туров: каталог, главная, направления, «Возможности», избранное
- Backend: обеспечить загрузку `cities` relation во всех контроллерах, отдающих туры

## Out of scope

- Изменения схемы БД (колонка `start_city` остаётся string, `city_tour` без изменений)
- Создание отдельного компонента `TourCard` (inline-карточки остаются)
- Изменение фильтрации по городам в каталоге
- Логика админки по управлению городами (чекбоксы остаются как есть)

## Constraints

- Max 5 файлов на шаг
- Все команды через Docker (см. spec-continuation.mdc)
- Используем существующий `RBadge` для чипсов городов
