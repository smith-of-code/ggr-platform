# Модуль: Туры возможностей (aportinity-tours)

## Статус: Реализован

## Цель

Отдельная публичная страница «Туры возможностей» с навигацией из шапки сайта. Первая версия — статическая (данные захардкожены), позднее будет управляться из админки.

## В scope

- Новый маршрут `GET /opportunity-tours` → `OpportunityToursController@index`
- Ссылка «Туры возможностей» в шапке (рядом с «Туры»), в мобильном меню и в footer
- Секция «Цифры проекта» — 4 счётчика: туров реализовано, гостей посетило, городов в прошлом году, городов в новом году
- Секция «Проекты программы» — 3 проекта (Старт в Атомград, Атомы вкуса, Лучшие люди Росатома): название (кликабельное, ссылка-заглушка) + краткое описание
- Секция «Видео туров» — слайдшоу видео (embed через ВК Видео iframe)
- Секция «Как принять участие» — 3 шага с описанием + кнопка «Личный кабинет»
- Секция «Счётчик эмоций» — статистика впечатлений (статические числа)
- Секция «Популярные туры» — слайдшоу карточек туров (статические данные, клик → заглушка)
- Секция «Социальные сети» — ссылки на соцсети проекта
- Секция «FAQ» — аккордеон с ответами на популярные вопросы
- Секция «Партнёры» — логотипы/названия партнёров

## Вне scope

- Управление контентом из админки (будет отдельная задача)
- Страницы отдельных проектов (Старт в Атомград и т.д.) — будет отдельная задача
- Реальные данные из БД (пока всё статическое)
- Загрузка/управление видео — пока embed-заглушки

## Ограничения

- Используем существующий `MainLayout.vue`
- Стилистика и UI-паттерны аналогичны `Home.vue` и `Tours/Index.vue`
- Tailwind CSS 4.x, Vue 3, Inertia.js
- Все данные захардкожены на фронте (не в контроллере), чтобы упростить будущий перенос в админку

## Маршруты

- `GET /opportunity-tours` → `OpportunityToursController@index` (публичный)
- Route name: `opportunity-tours.index`
