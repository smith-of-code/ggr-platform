# Модуль: Туры возможностей (aportinity-tours)

## Статус: Реализован

## Цель

Отдельная публичная страница «Туры возможностей» с навигацией из шапки сайта. Первая версия — статическая (данные захардкожены), позднее будет управляться из админки.

## В scope

- Новый маршрут `GET /opportunity-tours` → `OpportunityToursController@index`
- Ссылка «Туры возможностей» в шапке (рядом с «Туры»), в мобильном меню и в footer
- Секция «Цифры проекта» — счётчики (редактируемые из админки: значение + подпись)
- Секция «Проекты программы» — карточки Direction (управляются через админку направлений, `is_active` для скрытия)
- Секция «Видео туров» — слайдшоу видео (embed ВК/Рутьюб, добавление по ссылкам из админки)
- Секция «Как принять участие» — шаги с описанием + кнопка ЛК (ручное редактирование из админки)
- Секция «Счётчик эмоций» — статистика впечатлений (ручной ввод цифр из админки)
- Секция «Популярные туры» — слайдшоу карточек туров (выбор конкретных туров из каталога в админке)
- Секция «Социальные сети» — ссылки на соцсети (CRUD из админки: добавление/удаление, смена ссылок)
- Секция «FAQ» — аккордеон вопросов-ответов (CRUD из админки, поддержка гиперссылок в ответах)
- Секция «Партнёры» — кликабельные логотипы партнёров (CRUD из админки: логотип, название, URL)
- **Отдельная страница в админке** для редактирования всех блоков страницы, пункт в сайдбаре

## Вне scope

- Страницы отдельных проектов (Старт в Атомград и т.д.) — будет отдельная задача
- Счётчик эмоций: автоматический подсчёт (отдельное ТЗ, пока ручной ввод по согласованию с заказчиком)
- Добавление/удаление карточек в блоке «Цифры проекта» (запроса не было, но могут захотеть позже)

## Ограничения

- Используем существующий `MainLayout.vue` (публичная) и `AdminLayout.vue` (админка)
- Стилистика и UI-паттерны аналогичны существующим страницам
- Tailwind CSS 4.x, Vue 3, Inertia.js
- Данные хранятся в таблице `settings` (group `opportunity_tours`), JSON для массивов
- Используем существующие `Setting` модель и `SettingsService`

## Маршруты

- `GET /opportunity-tours` → `OpportunityToursController@index` (публичный)
- Route name: `opportunity-tours.index`
- `GET /admin/opportunity-tours-page` → `Admin\OpportunityToursPageController@index` (admin)
- `PUT /admin/opportunity-tours-page` → `Admin\OpportunityToursPageController@update` (admin)
- Route names: `admin.opportunity-tours-page.index`, `admin.opportunity-tours-page.update`

## Хранение данных

Группа `opportunity_tours` в таблице `settings`, ключи:
- `hero_title` — заголовок страницы (string)
- `hero_description` — описание под заголовком (string)
- `stats` — JSON массив `[{value, label}]`
- `emotions` — JSON массив `[{icon, count, label}]`
- `partners` — JSON массив `[{name, url, logo}]`
- `socials` — JSON массив `[{name, url, icon}]`
- `faq` — JSON массив `[{question, answer}]` (answer может содержать HTML)
- `videos` — JSON массив `[{title, embedUrl}]`
- `participation_steps` — JSON массив `[{title, description}]`
- `featured_tour_ids` — JSON массив ID туров `[1, 5, 12, ...]`
