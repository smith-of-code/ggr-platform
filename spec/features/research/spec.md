# Спецификация: research

## Цель

Заменить текущий CRUD-каталог исследований (`/research`, `/admin/research`) на статическую страницу с блочным контентом, редактируемым из админки, аналогично https://rosatom-travel.ru/research.

## В скоупе

- Удалить старый CRUD Research (модель, контроллеры, Vue-страницы, маршруты, связь в City)
- Вынести публичные маршруты рецептов в отдельный `RecipeController`
- Создать новую публичную страницу `/research` со следующими блоками:
  - **Hero** — заголовок, подзаголовок, текст описания, фоновое изображение и настраиваемый градиент (`from/via/to`) с цветом текста
  - **Общие задачи** — заголовок секции, список пунктов (текст)
  - **Пилотные города** — заголовок секции, карточки городов (название, регион, описание, изображение)
  - **Основа исследования** — заголовок секции, счётчики (число + подпись)
  - **Результаты** — заголовок, описание, ссылка на PDF (текст кнопки, URL)
  - **Города программы** — заголовок секции, карточки городов (название, регион)
- Создать в админке раздел «Исследования» (слева в меню, секция «Сайт») — единая страница редактирования контента по блокам
- Хранение через `settings` таблицу (group = `research_page`), паттерн OpportunityToursPage
- Seeder с данными из оригинальной страницы rosatom-travel.ru/research
- В админ-редакторе Hero поддержать выбор/загрузку фона через медиатеку (`ImageUploadCrop`) и поля: `hero_bg_color_from`, `hero_bg_color_via`, `hero_bg_color_to`, `hero_text_color`, `hero_bg_color_enabled`
- Для каждой контентной секции в админке должен быть доступен заголовок секции там, где он хранится в `research_page` (в т.ч. `tasks_title` для «Общие задачи»)

## Вне скоупа

- Миграция данных из старой таблицы `researches` (данные не нужны)
- Интерактивная карта (в новом дизайне не предусмотрена)
- Детальные страницы отдельных исследований (`/research/{slug}`)
- Страницы рецептов (остаются как есть, только переносятся в свой контроллер)

## Ограничения

- Паттерн хранения: `settings` (group/key/value), аналогично OpportunityToursPage
- Переиспользовать `DynamicList.vue` и `SectionHeader.vue` из `Admin/OpportunityToursPage/`
- Переиспользовать `SettingsService` для чтения/записи
- Максимум 5 файлов за шаг

## Открытые вопросы

- Нет (оригинал страницы изучен через WebFetch)
